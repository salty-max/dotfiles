---
name: Confirm before external actions
description: Always list planned actions and get approval before creating issues, PRs, or other external side effects
type: feedback
---

List out planned actions for user review before executing them, especially for external-facing operations like creating GitHub issues or PRs.

**Why:** User wants to review and approve the plan before anything is created.
**How to apply:** When asked to create multiple issues/PRs/etc., first present the list of what will be created and wait for confirmation.
