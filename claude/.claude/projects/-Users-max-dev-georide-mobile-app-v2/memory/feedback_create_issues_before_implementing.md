---
name: create_issues_before_implementing
description: Create sub-issues before starting implementation of large features
type: feedback
---

When a feature is large enough to be split into multiple PRs, create the sub-issues first before writing any code.

**Why:** The user wants to review and align on the breakdown before work begins. Jumping into code without agreed-upon sub-issues wastes effort if the scope needs adjustment.

**How to apply:** For any feature requiring multiple PRs, plan the breakdown, create GitHub issues for each piece, then wait for confirmation before implementing.
