Don't amend unrelated changes into existing commits. Create separate, focused commits instead.

**Why:** User prefers clean git history with each commit having a single purpose.

**How to apply:** When adding a new feature/rule/fix, always create a new commit rather than amending into an existing one, even if they're on the same branch.
